# Wand-Operationen

Dieses Recipe deckt Lesen, Modifizieren, Löschen und Klassifizieren von Wänden ab. **Wand-Erstellung ist im Archicad-MCP v29 nicht verfügbar** — siehe `../reference/mcp-conventions.md` § Capability-Tabelle. Wenn der User eine neue Wand braucht: zeichnet er sie manuell in Archicad, wir arbeiten dann an dieser Wand weiter.

## Inhaltsverzeichnis

1. [Umfang](#umfang)
2. [Erforderlicher Warm-up-Kontext](#erforderlicher-warm-up-kontext)
3. [Discovery-Anker](#discovery-anker)
4. [Typische Parameter](#typische-parameter)

## Umfang

- Lesen aller Wände einer Story oder des gesamten Projekts.
- Wand-Eigenschaften modifizieren (Höhe, Dicke, Geometrie, Layer, Material/Composite).
- Wand löschen (mit Hosted-Element-Pre-Check).
- Wand-Klassifikation lesen oder setzen.

Nicht abgedeckt: Wand erstellen (MCP-Gap).

## Worked Example — Wand modifizieren (Höhe ändern)

Wir ändern die Höhe der Test-Wand von 2.80m auf 3.20m.

**Sicherheits-Schritt:** Update → Confirm-Schleife. Vor dem Aufruf zeigen wir dem User:

```
Ich werde folgendes ändern:
- Wand <guid>  „<Wall-Name oder ‚unbenannt'>"   Höhe: 2.80m → 3.20m

Ausführen? (ja / nein / details / abbrechen)
```

Erst nach `ja`:

**Aufruf:**
```
mcp__archicad__archicad_call_tool(
  name="elements_set_details_of_elements",
  arguments={
    "port": <port>,
    "params": {
      "elementsWithDetails": [{
        "elementId": {"guid": "<test-wall-guid>"},
        "details": {
          "typeSpecificDetails": {"height": 3.20}
        }
      }]
    }
  }
)
```

[Erwartete Response erklären. SAFE-05 Element-ID-Threading erläutern: die ID wird für Folge-Operationen behalten.]

## Worked Example — Wand löschen (mit Hosted-Element-Check)

**SAFE-04 greift hier zwingend.** Bevor wir eine Wand löschen, listen wir die gehosteten Elemente (Fenster, Türen) auf.

**Schritt 1 — Hosted-Element-Pre-Check:**
```
mcp__archicad__archicad_call_tool(
  name="elements_get_connected_elements",
  arguments={
    "port": <port>,
    "params": {
      "elements": [{"elementId": {"guid": "<test-wall-guid>"}}],
      "connectedElementType": "Window"
    }
  }
)
```

[Erwartete Response zeigt das User-gezeichnete Fenster.]

**Schritt 2 — Confirm mit erweiterter Information:**

```
Ich werde folgendes löschen:
- Wand <guid>  (Story 0, gerade, 4m)
- Hostet folgende Elemente, die mit-gelöscht werden:
  - Fenster <window-guid>  (Sill 0.9m, 1.20×1.40m)

Ausführen? (ja / nein / details / abbrechen)
```

**Schritt 3 — Erst nach explizitem `ja` aufrufen:**
```
mcp__archicad__archicad_call_tool(
  name="elements_delete_elements",
  arguments={
    "port": <port>,
    "params": {
      "elements": [{"elementId": {"guid": "<test-wall-guid>"}}]
    }
  }
)
```

[Hinweis: Fenster wird durch Archicad automatisch mit-gelöscht, weil es auf der Wand hostet. Das ist Archicads natürliches Verhalten, nicht unsere Aufgabe.]

## Gotchas

- **Wand-Erstellung nicht via MCP v29** — User zeichnet manuell.
- **WallSettings.begCoordinate ist Coordinate2D** (kein Z), Höhe separat in `height`.
- **Modifikation behält Element-ID** — keine neue ID nach Update.
- **Layer-Index ist eine `number`-1.0 (float, 1-basiert)**, nicht 0-basiert. Aus Phase-2-Layer-Listing übernommen.
- **Hosted-Element-Mit-Delete** ist Archicad-Verhalten, nicht MCP — das Pre-Check warnt den User trotzdem.
